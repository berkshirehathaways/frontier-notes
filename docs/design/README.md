# Design Docs

프론티어노트의 디자인 문서는 책임별로 나뉩니다.

- `../../DESIGN.md` — 색, 타이포그래피, 간격, 컴포넌트 원칙
- `content-hierarchy.md` — 전역, 페이지, 상세, 목록의 상위 정보 위계
- `content-type-system.md` — Issues, Field Notes, Essays, Reports, Systems, Interviews, People, Tools별 카드/상세/모바일 포맷

UI를 바꿀 때는 어떤 규칙을 바꿨는지에 따라 문서도 함께 갱신합니다. 토큰이나 재사용 컴포넌트 원칙은 `DESIGN.md`, 화면의 상위 정보 위계는 `content-hierarchy.md`, 콘텐츠 타입별 표현 계약은 `content-type-system.md`가 담당합니다.
